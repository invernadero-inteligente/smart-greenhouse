🚀 PRÓXIMOS PASOS
═══════════════════════════════════════════════════════════════════════

1. ✅ Configurar base de datos PostgreSQL
   $ createdb invernadero_db

2. ✅ Configurar MongoDB
   $ docker run -d -p 27017:27017 --name mongodb mongo:latest

3. ✅ Copiar aplicación properties
   $ cp src/main/resources/application-templates.properties \
        src/main/resources/application.properties

4. ✅ Compilar proyecto
   $ mvn clean compile

5. ✅ Ejecutar aplicación
   $ mvn spring-boot:run

6. ✅ Probar endpoints
   GET http://localhost:8080/api/v1/invernaderos

💡 TIPS PARA EL EQUIPO
═══════════════════════════════════════════════════════════════════════

🎯 Para crear una nueva entidad (ej: Sensor):
   1. Copiar patrón de Invernadero (está comentado para copiar)
   2. Cambiar nombres: Invernadero → Sensor
   3. Agregar propiedades específicas de Sensor
   4. Crear Repository, Service, Mapper, DTOs, Controller

🔄 Flujo de Desarrollo:
   git checkout develop
   git checkout -b feature/is/nombre-funcionalidad
   [Hacer cambios]
   git commit -m "feat(is): descripción"
   git push origin feature/is/nombre-funcionalidad
   [Abrir Pull Request]

📋 Usar Templates:
   - InvernaderoController → Template para otros controllers
   - InvernaderoService → Template para otros servicios
   - InvernaderoRepository → Template para custom queries
   - InvernaderoServiceImpl → Template para implementaciones
